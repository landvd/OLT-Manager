# Feature Prompt

你正在 OLT Manager 仓库中实现一个新功能。请先按 `AGENTS.md` 的读取顺序理解项目。

## 必须确认

- 这个功能是否属于 `docs/requirements/PRD.md` 的 MVP。
- 是否涉及 OLT 设备访问。
- 是否保持只读。
- 是否需要更新 API、数据库、时序图或 ADR。
- 是否会接触真实 IP、community、账号、密码或现场台账。

## 工作流程

1. 检查当前分支和未提交改动。
2. 阅读相关文档。
3. 写出最小设计和验证标准。
4. 先补测试或 fixture，再实现。
5. 保持改动小而明确。
6. 运行构建和相关检查。
7. 更新 `CHANGELOG.md`。

## 禁止

- 不要加入 OLT 写操作。
- 不要暴露任意 CLI。
- 不要提交敏感运行数据。
- 不要顺手重构无关代码。
