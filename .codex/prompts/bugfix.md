# Bugfix Prompt

你正在 OLT Manager 仓库中修复一个问题。请用可复现证据驱动，不要凭感觉改。

## 步骤

1. 描述现象和影响范围。
2. 找到最小复现路径。
3. 判断是前端状态、API、SNMP 解析、SQLite、Telnet 输出清洗还是环境问题。
4. 增加最小测试、fixture 或手工验证命令。
5. 做最小修复。
6. 重新运行验证。
7. 更新 `CHANGELOG.md` 或 `EXPERIMENTS.md`。

## 设备安全

- 排障时只允许读取。
- 不使用 `snmpset`。
- 不执行任意 Telnet/SSH。
- 不把真实凭据写入日志、测试或文档。

## 输出要求

- 说明根因。
- 说明修改文件。
- 说明验证命令和结果。
- 说明剩余风险。
